# Как выложить сайт на GitHub Pages (бесплатно, навсегда)

Всё делается в браузере — ничего устанавливать не нужно.

---

## Шаг 1. Создайте аккаунт GitHub
1. Откройте **github.com** → **Sign up**.
2. Зарегистрируйтесь (почта, пароль, имя пользователя). Бесплатно, карта не нужна.
3. Подтвердите почту.

> Имя пользователя запомните — оно попадёт в адрес сайта.
> Например, если логин `andrey`, сайт будет на `andrey.github.io/...`

---

## Шаг 2. Создайте репозиторий (это «папка» для сайта)
1. Справа вверху нажмите **+** → **New repository**.
2. **Repository name** — впишите, например: `site` (или `nutrition`).
3. Поставьте **Public** (для бесплатного Pages обязательно публичный).
4. Нажмите **Create repository**.

> Хотите адрес без хвоста, вида `andrey.github.io` (без `/site`)?
> Тогда назовите репозиторий ровно так: `ВАШ-ЛОГИН.github.io`
> (например `andrey.github.io`). Это необязательно.

---

## Шаг 3. Загрузите файлы сайта
1. На странице нового репозитория нажмите ссылку **uploading an existing file**
   (или кнопка **Add file → Upload files**).
2. Откройте папку `site` у себя на компьютере, **выделите ВСЕ файлы внутри**
   (index.html, styles.css, app.js, image-slot.js, tweaks-panel.jsx, tweaks.jsx)
   и перетащите их в окно браузера.

   ⚠️ Важно: перетаскивайте САМИ ФАЙЛЫ, а не папку `site` целиком —
   `index.html` должен оказаться в корне репозитория.
3. Внизу нажмите зелёную кнопку **Commit changes**.

---

## Шаг 4. Включите GitHub Pages
1. В репозитории сверху откройте вкладку **Settings**.
2. Слева в меню найдите **Pages**.
3. В блоке **Build and deployment → Source** выберите **Deploy from a branch**.
4. Ниже в **Branch** выберите **main** и папку **/ (root)** → **Save**.
5. Подождите 1–2 минуты и обновите страницу.

Вверху появится зелёная плашка с адресом:
**`https://ВАШ-ЛОГИН.github.io/site/`**

Это и есть постоянная ссылка на ваш сайт — можно слать кому угодно. 🎉

---

## Как потом вносить изменения
1. Я даю вам обновлённые файлы.
2. В репозитории: **Add file → Upload files**, перетаскиваете новые версии
   (старые заменятся), → **Commit changes**.
3. Через минуту сайт обновится по той же ссылке.

---

## Если что-то не открылось
- Адрес активируется не мгновенно — подождите 1–2 минуты после **Save**.
- Проверьте, что репозиторий **Public** и что `index.html` лежит в корне (не внутри подпапки).
- Логин в адресе — маленькими буквами.

Застрянете на любом шаге — пришлите скриншот, подскажу.

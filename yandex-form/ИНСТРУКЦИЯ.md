# Подключение формы через Yandex Cloud Postbox

Цель: заявка с формы на сайте → функция в Yandex Cloud → письмо тебе на почту.
Вся цепочка на российской инфраструктуре. Дизайн формы не меняется.

```
Браузер посетителя ─► functions.yandexcloud.net (РФ) ─► Yandex Cloud Postbox ─► 📧 тебе на почту
```

Разовая настройка ~30–60 минут. Делается один раз. Что нужно: аккаунт в Yandex Cloud
с привязанной картой (для верификации, списаний в пределах бесплатных лимитов не будет).

---

## Шаг 0. Аккаунт и платёжный аккаунт
1. Зайди на **console.yandex.cloud**, войди под Яндекс-аккаунтом.
2. Создастся **облако** и **каталог** (folder) — по умолчанию `default`. Запомни, в каком
   каталоге работаешь: сервисный аккаунт и адрес отправителя должны быть **в одном каталоге**.
3. Привяжи платёжный аккаунт (карта для верификации). Бесплатные лимиты: функции — 1 млн
   вызовов/мес, Postbox — первые 2000 писем/мес бесплатно.

---

## Шаг 1. Postbox: подтвердить адрес отправителя (identity)
Письма нельзя слать «с потолка» — нужен подтверждённый адрес-отправитель.

1. В консоли открой сервис **Postbox** → **Создать → Адрес (identity)**.
2. Укажи адрес, **с которого** будут уходить письма, например `noreply@твойдомен.ru`.
   - Если своего домена нет — можно подтвердить **отдельный email-адрес** (адрес любого уровня).
     Проще всего, если у тебя есть домен: тогда подтверждается весь домен.
3. В блоке DKIM выбери **внутреннюю подпись** (signing-type internal) — Яндекс сам подпишет письма.
4. Нажми **Создать**. Postbox покажет:
   - либо **проверочное письмо** на этот адрес (перейди по ссылке внутри),
   - либо **DNS-записи** (DKIM/верификация) — добавь их у регистратора домена.
5. Дождись статуса **Подтверждён / Verified**. Без этого письма уходить не будут.

> ⚠️ `SENDER` в функции = этот подтверждённый адрес.
> `RECIPIENT` (куда падают заявки) — твоя обычная почта, её подтверждать не нужно.

---

## Шаг 2. Сервисный аккаунт с правом отправки
Функция будет слать письма «от имени» сервисного аккаунта.

1. Консоль → **Сервисные аккаунты** → **Создать**. Имя любое, напр. `sa-postbox`.
2. Открой его и выдай роль **`postbox.sender`** (на каталог, где создан адрес из Шага 1).
   - Важно: сервисный аккаунт и адрес-отправитель — **в одном каталоге**, иначе будет ошибка.

Никаких ключей вручную создавать не нужно — функция получит токен этого аккаунта сама.

---

## Шаг 3. Создать функцию
1. Консоль → **Cloud Functions** → **Создать функцию**. Имя, напр. `lead-form`.
2. Внутри функции → **Создать редактор/версию**:
   - **Среда выполнения (Runtime):** `nodejs18` (или новее).
   - **Способ:** редактор кода (или ZIP).
   - Создай два файла и вставь содержимое из этой папки:
     - `index.js`     ← код функции (файл `yandex-form/index.js`)
     - `package.json` ← файл `yandex-form/package.json`
   - **Точка входа (Entrypoint):** `index.handler`
3. В параметрах версии:
   - **Сервисный аккаунт:** выбери `sa-postbox` из Шага 2. ← без этого письма не уйдут.
   - **Переменные окружения** (Environment variables):
     - `SENDER`    = подтверждённый адрес из Шага 1 (напр. `noreply@твойдомен.ru`)
     - `RECIPIENT` = твоя почта для заявок (напр. `andrey@yandex.ru`)
   - Таймаут можно оставить по умолчанию (например 10 сек), память 128 МБ — хватит.
4. Нажми **Создать версию**.

---

## Шаг 4. Сделать функцию публичной
Чтобы браузер посетителя мог её вызвать.

1. На странице функции включи **«Публичная функция»** (Make function public /
   роль `functions.functionInvoker` для `allUsers`).
2. Скопируй **ссылку для вызова** вида:
   ```
   https://functions.yandexcloud.net/d4xxxxxxxxxxxxxxxxxx
   ```

---

## Шаг 5. Вставить ссылку в сайт
1. Открой файл **`sections.js`** в проекте.
2. Найди строку (в самом начале блока ФОРМА):
   ```js
   var LEAD_ENDPOINT='ВСТАВЬ_СЮДА_ССЫЛКУ_ФУНКЦИИ';
   ```
3. Замени на свою ссылку из Шага 4:
   ```js
   var LEAD_ENDPOINT='https://functions.yandexcloud.net/d4xxxxxxxxxxxxxxxxxx';
   ```
4. Сохрани. Готово — форма шлёт заявки на почту.

> Пока ссылка не вставлена, форма ведёт себя как раньше (показывает «Спасибо»),
> но в консоль браузера пишет предупреждение, что заявка никуда не ушла.

---

## Шаг 6. Проверка
1. Открой сайт, заполни форму, отправь.
2. Должно прийти письмо на `RECIPIENT` с темой «Заявка с сайта — Имя».
3. Если письма нет — проверь по списку «Если не работает» ниже.

---

## Если не работает — что смотреть
- **Письмо не пришло, в логах функции `No service account token`** — к функции не привязан
  сервисный аккаунт (Шаг 3, пункт «Сервисный аккаунт»).
- **Ошибка `Postbox 403 / not verified`** — адрес `SENDER` ещё не подтверждён (Шаг 1)
  или сервисному аккаунту не выдана роль `postbox.sender` (Шаг 2).
- **Ошибка про разные каталоги** — адрес-отправитель и сервисный аккаунт в разных каталогах
  (folder). Создай их в одном.
- **В браузере ошибка CORS** — это значит запрос дошёл, но проверь, что функция публичная (Шаг 4)
  и в коде остались CORS-заголовки (они уже есть в `index.js`).
- **Логи функции:** Cloud Functions → твоя функция → вкладка «Логи». Там видно текст ошибки.
- Первые письма иногда попадают в **Спам** — проверь папку, отметь «не спам».

---

## Лимиты и деньги
- Функция: 1 000 000 вызовов/мес бесплатно — для лендинга это практически бесконечно.
- Postbox: первые 2000 писем/мес бесплатно.
- Платёжный аккаунт нужен для верификации; списаний не будет, пока не выйдешь за лимиты.
